## Описание проекта

Кроссплатформенное десктопное приложение с графическим интерфейсом (GUI) для генерации криптографически стойких случайных паролей.
Приложение позволяет пользователю гибко настраивать параметры генерации и вести историю созданных паролей.

### Основные возможности:
*   **Настройка длины:** Ползунок для выбора длины пароля от 4 до 32 символов.
*   **Выбор символов:** Возможность включать/выключать использование цифр (0-9), букв латинского алфавита (a-zA-Z) и специальных символов (!@#$%^&*).
*   **История:** Все сгенерированные пароли сохраняются в локальный файл `history.json`. История отображается в виде таблицы и сохраняется между запусками программы.
*   **Копирование:** Возможность быстрого копирования сгенерированного пароля в буфер обмена.
*   **Контроль версий:** Проект управляется с помощью системы Git и размещен на платформе [GitHub](https://github.com).

## Примеры использования и тесты

Ниже приведены примеры сценариев использования приложения для проверки его работоспособности.

### Сценарий 1: Генерация простого цифрового PIN-кода

1.  Установите ползунок длины на значение `4`.
2.  Отметьте галочкой только чекбокс **"Цифры"**. Снимите галочки с "Буквы" и "Спецсимволы".
3.  Нажмите кнопку **"Сгенерировать пароль"**.
4.  **Ожидаемый результат:** В поле ввода появится строка длиной ровно 4 символа (например: `8392`), состоящая только из цифр. Пароль должен добавиться в таблицу истории.
5.  Закройте программу и запустите снова. Пароль должен остаться в истории.
6.  *Проверка:* `len(generated_password) == 4` и `generated_password.isdigit() == True`.

### Сценарий 2: Генерация сложного пароля максимальной длины

1.  Установите ползунок длины на максимальное значение `32`.
2.  Отметьте все три чекбокса: **"Цифры"**, **"Буквы"**, **"Спецсимволы"**.
3.  Нажмите кнопку генерации.
4.  **Ожидаемый результат:** В поле ввода появится строка длиной ровно 32 символа. Строка должна содержать как минимум одну цифру, одну букву верхнего регистра и один спецсимвол.
5.  *Проверка:* `len(generated_password) == 32`.
6.  *Проверка состава:*
    ```python
    assert any(c.isdigit() for c in generated_password)   # Есть цифры?
    assert any(c.isalpha() for c in generated_password)   # Есть буквы?
    assert any(c in string.punctuation for c in generated_password) # Есть спецсимволы?
    ```
7.  Нажмите кнопку **"Копировать"**. Проверьте содержимое буфера обмена — оно должно совпадать с полем ввода.
